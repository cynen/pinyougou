
商城项目:


POM打包方式： 主要是做依赖的版本管理。【控制统一依赖】

JAR打包方式： 主要是作为各个模块的依赖，【存在依赖传递】 

WAR打包方式：将模块作为一个应用程序，独立的发布运行。【无论是否存在前端页面】

WAR注意区分SpringBoot中的jar打包方式。



使用方式:

1.启动对应的service项目

2.启动对应的web项目.




# Notice:

pages 文件夹是静态页面生成后需要引用的文件集合.

目录结构:
pages
|--css
|--fonts
|--img
|--js
|--plugins
